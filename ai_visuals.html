Phase 2
Story Intelligence Engine
Fast rule-based topic, urgency + region detection
Goal
Build a cheap, fast classifier that takes a headline string and returns structured metadata without touching the LLM. This runs on every article before anything expensive fires.
Three sub-engines to build
Topic Engine
Keyword maps + regex. Input: headline string. Output: { topic: "crypto" | "ai" | "tradfi" | "cybercrime" | "regulation" | "startup-vc" }. Build a keyword dictionary per topic. Use OR-matching, not AI.
Urgency Engine
Pattern matching on trigger words: "crash", "hack", "SEC", "ETF approval", "breaking", "emergency". Output: { urgency: "breaking" | "high" | "medium" | "low" }. Also parse publication timestamp delta for recency signals.
Region Detector
Named entity + geo-keyword matching. Flags Africa, Asia, Europe, US. Outputs { region: "africa" | "global" | "us" | "asia" }. African region flag activates Afrofuturism visual track.

Phase 3
LLM Scene Planner
Cinematic direction, symbolism + emotional framing via Claude API
Goal
Call the LLM only for what rule-based systems cannot do: narrative abstraction, emotional tone, symbolism selection, cinematic direction. This layer outputs structured JSON — not image prompts. The LLM never writes the final prompt directly.
Input to the LLM
Pass the headline + narrative metadata from Phase 2 + relevant Visual Bible rules for the detected category. Include a strict system prompt: "You are a cinematic art director. Return only valid JSON. Never write image generation prompts."

Phase 4
Prompt Composer + ComfyUI Injector
Scene JSON → diffusion prompt → ComfyUI workflow execution
Goal
Convert the scene planner's JSON into a final positive/negative diffusion prompt string using deterministic composition logic — no LLM. Then inject those strings into the correct ComfyUI workflow JSON and submit to the API.
Prompt Composer logic
Build ordered prompt segments: environment → subjects → lighting → camera → style modifiers → quality boosters → negative rules. Each segment pulls from Visual Bible JSON. Compose them with comma separation. Run a de-duplication pass. Cap at 200 tokens positive, 80 tokens negative.
Workflow Router — routing table
urgency === "breaking" → workflows/breaking/
  model: SDXL Lightning
  steps: 20, cfg: 5, res: 768px
  SLA: 2–10 seconds

story_type === "feature" → workflows/premium/
  model: JuggernautXL v9
  steps: 40, cfg: 7, res: 1024px
  + IPAdapter + ControlNet depth
  SLA: 30–90 seconds

topic === "cybercrime" → workflows/cybercrime/
  + glitch overlay node
  + corrupted UI node

region === "africa" → workflows/afrofuturism/
  + warm LUT post-process
  + regional LoRA (if trained)
ComfyUI Injector function
Read the workflow JSON from disk. Inject: node["6"].inputs.text (positive prompt), node["7"].inputs.text (negative), width/height into the KSampler node, steps, cfg, seed. Return the mutated workflow object. Submit via POST /prompt to the ComfyUI API. Poll websocket for completion.
Node IDs are workflow-specific Map them per workflow type Seed = random UUID int

Phase 5
Redis + BullMQ Orchestration
Async queue infrastructure, priority lanes + GPU workers
Goal
Generation must never be synchronous. Every image request enters a priority queue. GPU workers pull from that queue. This decouples the API layer from generation time and enables horizontal scaling.
Queue architecture
Queue name Priority
─────────────────────── ────────
breaking-priority 10 (highest)
premium-priority 7
thumbnail-fast 5
upscale 4
quality-check 3
retry-failed 1 (lowest)
GPU Worker responsibilities
Each worker: (1) pulls job from BullMQ, (2) calls workflowRouter to get workflow path, (3) calls injectWorkflow with scene data, (4) submits to ComfyUI API, (5) monitors websocket until execution_complete, (6) fetches image binary from ComfyUI /view endpoint, (7) uploads to S3/R2, (8) emits job completion event with CDN URL and metadata.

Phase 6
Automated Quality Scoring
CLIP + aesthetic ranker + artifact detection before publish
Goal
No generated image reaches the CMS without passing automated quality gates. This runs as a BullMQ quality-check job immediately after generation and before upscaling or delivery.
Quality pipeline steps
Step 1
CLIP scoring — compute cosine similarity between the final prompt and the generated image embedding. Reject if score < 0.75.
Step 2
LAION Aesthetic Predictor — run the LAION aesthetic model (Python subprocess or REST). Reject if aesthetic score < 6.5.
Step 3
Artifact detection — check blur (Laplacian variance), text artifact presence (lightweight OCR pass), and extreme brightness/darkness thresholds.
Step 4
Brand alignment — compute color distribution match against the expected palette for this story type. Reject if dominant hue is outside the brand palette range.
Score output contract
{
"clip_score": 0.92,
"aesthetic_score": 8.7,
"artifact_score": 0.03,
"brand_alignment": 0.88,
"approved": true,
"rejection_reason": null
}
On rejection
Automatically re-queue with a new random seed. Max 3 retries. On 3rd failure: flag for human review, serve the best-scoring candidate with a review watermark. Never silently fail.
Python subprocess for CLIP/LAION Node wraps via child_process Max 3 retries

Phase 7
Thumbnail Intelligence
CTR-optimized crops, saliency detection + platform variants
Goal
Hero images and thumbnails are generated separately. A thumbnail is not a cropped hero — it's a different composition optimized for one focal point, high contrast, and emotional clarity at small sizes.
Two generation tracks
Track A — Dedicated thumbnail generation
Run a separate 16-step ComfyUI job at 512px with a thumbnail-specific scene plan (simplified composition, single subject, high contrast rules). This is the preferred track for new articles.
Track B — Smart crop from hero
Use saliency detection (e.g. basnet or u2net) to find the dominant focal point in the hero image. Apply focal-aware smart crop. Only use this as a fallback.
